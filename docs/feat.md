# feat.md — 未来优化方向

> 基于当前代码现状整理的优化路线图，按类别分组，标注优先级（P0 高 / P1 中 / P2 低）与用户决策。
> 每条均说明现状、问题与建议改动。

---

## 一、搜索体验

### 1. 升级为 FlexSearch 全文检索 — P0 ✅ 要做
- **现状**：[search.js](iris/js/search.js) 虽引入了 `flexsearch.bundle.js`，但实际使用的是 `simpleSearch()` 的 `includes` 子串匹配，未做分词与模糊匹配，中文长查询命中差。
- **改动**：用 FlexSearch 构建标题+正文+路径的复合索引，支持中文分词与模糊匹配，结果更准。

### 2. 搜索结果关键词高亮 — P1 ✅ 要做
- **现状**：结果只显示纯文本预览，无法看出匹配位置。
- **改动**：在 title/preview 中高亮命中片段（`<mark>`），便于快速定位。

### 3. 搜索范围切换 — P1 ✅ 要做
- **现状**：标题与正文混合打分，无法单独按标题检索。
- **改动**：增加「仅标题 / 全文」切换，标题命中优先。

### 4. 搜索路径硬编码修复 — P0 ✅ 要做
- **现状**：[search.js#L142](iris/js/search.js#L142) 点击结果时硬编码拼接 `'docs/' + result.path`，非 `docs/` 下的文档会 404。
- **改动**：直接使用 `doc.path`（已在索引中），移除前缀拼接。

## 二、阅读体验

### 5. 目录树（Index）可折叠与高亮 — ❌ 不做
- **决策**：保持当前扁平列表+左边界高亮现状。

### 6. 阅读进度增强 — ❌ 不做
- **决策**：保持仅顶部一条整体进度条。

### 7. 上一篇 / 下一篇导航 — ⚠️ 待确认
- **现状**：`getAdjacentFiles` 逻辑已实现，但文末未渲染导航 UI。
- **决策**：用户反馈「不是已经有这个功能了吗」，待确认是否补全 UI。

### 8. 面包屑交互增强 — ❌ 不做
- **决策**：保持现状，仅展示路径。

## 三、主题与外观

### 9. 跟随系统明暗主题 — ❌ 不做
- **决策**：保持手动选择主题。

### 10. 可视化主题自定义 — P1 ✅ 要做
- **现状**：仅支持自定义 CSS URL。
- **改动**：设置面板提供取色器调整主色/强调色，保存到 localStorage。

### 11. 字体/字号/行高可调 — ❌ 不做
- **决策**：保持固定字体样式。

## 四、性能与加载

### 12. 首屏性能优化 — P1 ✅ 要做
- **现状**：文档与数据全量加载，无懒加载。
- **改动**：路由按需加载文档、预构建数据压缩、关键 CSS 内联，降低 LCP。

### 13. PWA 离线能力增强 — P1 ✅ 要做
- **现状**：[sw.js](sw.js) 为基础 Service Worker。
- **改动**：预缓存文档与静态资源，离线可读已访问文档；资源更新时提示用户刷新。

### 14. 构建脚本增量构建 — ❌ 不做
- **决策**：保持每次全量重建。

---

## 五、内容渲染

### 15. 代码块增强 — P1 ✅ 要做
- **现状**：代码块仅高亮，无交互。
- **改动**：每块增加复制按钮、语言标签、可选行号、长代码折叠。

### 16. 图片画廊/灯箱增强 — P2 ✅ 要做
- **现状**：图片有懒加载与错误降级，但无灯箱。
- **改动**：点击图片打开灯箱，支持缩放、拖动、键盘翻页。

## 七、SEO 与分享

### 19. 动态 SEO meta 与 Open Graph — P2 ✅ 要做
- **现状**：纯静态站点，所有文档共享同一组 meta。
- **改动**：每篇文档动态生成 `title` / `description` / `og:` 标签，便于社交分享与搜索引擎收录。

### 20. 标题锚点分享（复制直达链接） — P2 ✅ 要做
- **现状**：Hash 路由支持文档级 URL，但标题无锚点复制。
- **改动**：标题悬浮出现「复制链接」按钮，分享直达章节。
