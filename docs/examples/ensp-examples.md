# 华为 eNSP 拓扑渲染

本站点支持在 Markdown 文档中嵌入华为 eNSP 拓扑图。将 `.topo` 或 `.zip` 压缩包放入 `iris/data/ensp/raw/` 目录，构建脚本会自动解压、解析并生成 JSON，前端用 Cytoscape.js 渲染为交互式拓扑图。

## 基本用法

使用 `@[ensp](文件名)` 嵌入语法引用预处理好的拓扑 JSON，需要用 `ensp` 代码块包裹（即三反引号 + `ensp` 语言标记）：

```markdown
@[ensp](filename)
```

> **提示**：如果文件名包含空格，引用时请使用 URL 编码（空格替换为 `%20`）。例如 `@[ensp](my%20topo)`。

## 示例拓扑

### OSPF / IS-IS 排错综合实验

## 支持的输入格式

eNSP 解析器支持以下两种输入方式：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 单个拓扑文件 | `.topo` | 直接解析 XML 格式的拓扑描述 |
| 工程压缩包 | `.zip` | 自动解压 ZIP，内部需包含 `.topo` 和按 UUID 命名的设备目录 |

### ZIP 压缩包内部结构

```
my-topo.zip
├── my-topo.topo                  # 拓扑描述文件
├── {UUID-1}/                     # 设备 1 目录（按 UUID 命名）
│   └── vrpcfg.zip                # 设备配置（VRP 命令）
├── {UUID-2}/
│   └── vrpcfg.zip
└── ...
```

每个设备目录名对应 `.topo` 中 `<dev id="UUID">` 的设备 ID，内部 `vrpcfg.zip` 包含该设备的 VRP 配置（`vrpcfg.cfg`）。

## 交互功能

- **缩放平移**：鼠标滚轮缩放，拖拽平移画布
- **节点拖拽**：拖动设备节点重新排列
- **hover 高亮**：鼠标悬停连线显示接口、线缆和所属网段
- **点击详情**：点击设备节点弹出右侧抽屉，查看接口表 / VRP 配置 / VLAN / ACL / 路由表
- **搜索过滤**：在搜索框输入设备名 / IP / 类型，匹配设备高亮
- **布局切换**：eNSP 原始坐标 ↔ 力导向自动布局
- **网格背景**：可切换点阵网格 / 透明背景
- **导出**：支持导出 PNG 图片 / JSON 数据 / Markdown 表格

## 线缆类型颜色编码

| 颜色 | 线缆类型 |
|------|----------|
| 橙色 | 串口线 (Serial) |
| 蓝色 | 双绞线 (Copper) |
| 紫色 | 光纤 (Fiber) |
| 绿色 | 控制台线 (Console) |

## 设备类型图标

支持以下设备类型的图标渲染：AR 路由器、S 系列交换机、PC、服务器、防火墙、AP、Cloud 等。

## 工作流程

1. 将 `.topo` 或 `.zip` 文件放入 `iris/data/ensp/raw/`
2. 本地运行 `python iris/scripts/ensp/main.py` 或推送到 GitHub 触发 CI
3. 脚本自动解压（ZIP）→ 解析 XML → 读取 `vrpcfg.zip` 配置 → 输出 JSON
4. 在 Markdown 中用 `@[ensp](文件名)` 引用

### 文件命名建议

- 推荐使用英文、数字、连字符（`-`）和下划线（`_`）
- 避免空格和特殊字符
- 含空格的文件名引用时需用 `%20` 编码

## 解析的信息

每个设备可查看以下信息：

- **接口表**：接口名、IP、掩码、状态、速率、双工、描述
- **配置**：完整 `vrpcfg.cfg`（VRP 命令），按 interface 分块折叠，VRP 语法高亮
- **VLAN**：VLAN ID 和名称
- **ACL**：访问控制列表规则
- **路由表**：直连路由、静态路由、OSPF 路由（从配置推算）
