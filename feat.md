# feat.md — 未来优化方向

> 基于当前代码现状整理的优化路线图，按类别分组，标注优先级（P0 高 / P1 中 / P2 低）。
> 每条均说明现状、问题与建议改动。

---

## 一、搜索体验

### 1. 升级为 FlexSearch 全文检索 — P0
- **现状**：[search.js](iris/js/search.js) 虽引入了 `flexsearch.bundle.js`，但实际使用的是 `simpleSearch()` 的 `includes` 子串匹配，未做分词与模糊匹配，中文长查询命中差。
- **改动**：用 FlexSearch 构建标题+正文+路径的复合索引，支持中文分词与模糊匹配，结果更准。

### 2. 搜索结果关键词高亮 — P1
- **现状**：结果只显示纯文本预览，无法看出匹配位置。
- **改动**：在 title/preview 中高亮命中片段（`<mark>`），便于快速定位。

### 3. 搜索范围切换 — P2
- **现状**：标题与正文混合打分，无法单独按标题检索。
- **改动**：增加「仅标题 / 全文」切换，标题命中优先。

### 4. 搜索路径硬编码修复 — P0
- **现状**：[search.js#L142](iris/js/search.js#L142) 点击结果时硬编码拼接 `'docs/' + result.path`，非 `docs/` 下的文档会 404。
- **改动**：直接使用 `doc.path`（已在索引中），移除前缀拼接。

---

## 二、阅读体验

### 5. 目录树（Index）可折叠与高亮 — P1
- **现状**：目录为扁平列表，仅靠左边界颜色标识当前章节。
- **改动**：支持二级标题折叠/展开、当前章节自动展开、点击平滑滚动并高亮，与文件树交互一致。

### 6. 阅读进度增强 — P2
- **现状**：仅顶部一条整体进度条。
- **改动**：增加章节级进度与预计剩余阅读时间。

### 7. 上一篇 / 下一篇导航 — P1
- **现状**：`getAdjacentFiles` 已实现但未在文末展示。
- **改动**：文末渲染上一篇/下一篇卡片，便于连续阅读。

### 8. 面包屑交互增强 — P2
- **现状**：面包屑已存在但交互较弱。
- **改动**：每级可点击跳转到对应目录视图。

---

## 三、主题与外观

### 9. 跟随系统明暗主题 — P1
- **现状**：主题仅手动选择。
- **改动**：增加 `auto` 选项，依据 `prefers-color-scheme` 自动切换明暗主题。

### 10. 可视化主题自定义 — P2
- **现状**：仅支持自定义 CSS URL。
- **改动**：设置面板提供取色器调整主色/强调色，保存到 localStorage。

### 11. 字体/字号/行高可调 — P2
- **现状**：字体固定。
- **改动**：设置面板提供正文字号、字体族、行高调节并持久化。

---

## 四、性能与加载

### 12. 首屏性能优化 — P1
- **现状**：文档与数据全量加载，无懒加载。
- **改动**：路由按需加载文档、预构建数据压缩、关键 CSS 内联，降低 LCP。

### 13. PWA 离线能力增强 — P1
- **现状**：[sw.js](sw.js) 为基础 Service Worker。
- **改动**：预缓存文档与静态资源，离线可读已访问文档；资源更新时提示用户刷新。

### 14. 构建脚本增量构建 — P2
- **现状**：[build-file-tree.js](iris/scripts/build-file-tree.js) 与搜索索引每次全量重建。
- **改动**：按 git 变更增量更新 `file-tree.json` / `search-index.json`，并增加死链校验。

---

## 五、内容渲染

### 15. 代码块增强 — P1
- **现状**：代码块仅高亮，无交互。
- **改动**：每块增加复制按钮、语言标签、可选行号、长代码折叠。

### 16. 图片画廊/灯箱增强 — P2
- **现状**：图片有懒加载与错误降级，但无灯箱。
- **改动**：点击图片打开灯箱，支持缩放、拖动、键盘翻页。

---

## 六、可访问性与交互

### 17. 可访问性（a11y）系统补齐 — P1
- **现状**：缺少 `aria` 标签、键盘导航、焦点可见样式；部分对比度不足。
- **改动**：补全 aria 角色/标签、键盘可操作、焦点环可见、对比度修正。

### 18. 全局快捷键 — P2
- **现状**：仅有 Esc 关闭侧栏。
- **改动**：`/` 聚焦搜索、`j/k` 上下篇、`t` 切换目录、`g` 跳转主题等。

---

## 七、SEO 与分享

### 19. 动态 SEO meta 与 Open Graph — P1
- **现状**：纯静态站点，所有文档共享同一组 meta。
- **改动**：每篇文档动态生成 `title` / `description` / `og:` 标签，便于社交分享与搜索引擎收录。

### 20. 文档级分享与锚点链接 — P2
- **现状**：Hash 路由支持文档级 URL。
- **改动**：标题支持「复制锚点链接」，分享直达章节。

---

## 八、工程与质量

### 21. 核心模块单元测试 — P1
- **现状**：无测试（`jsdom` 已在 devDependencies 未使用）。
- **改动**：为路由/搜索/文件树构建等核心模块加单元测试，并在 CI 中运行。

### 22. 界面国际化（i18n） — P2
- **现状**：界面纯中文硬编码。
- **改动**：提取文案为字典，支持中英文切换。

### 23. 静态评论系统 — P2
- **现状**：纯阅读站点，无互动。
- **改动**：接入 Giscus / Utterances（基于 GitHub Discussions/Issues）实现静态评论。

### 24. 调试日志清理 — P2
- **现状**：[search.js](iris/js/search.js) 等模块含大量 `console.log` 调试输出，生产环境未清理。
- **改动**：统一由 `debug.js` 控制，非调试模式静默。
